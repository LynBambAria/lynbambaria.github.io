一、通信协议

1. GPIO

1.1 概念

GPIO 是嵌入式系统中用于与外部设备交互的通用接口

1.2 功能

  • 数字输入：读取外部信号，如按键状态、传感器数据等
  • 数字输出：控制外设工作，如LED、BEEP、继电器等
  • 模拟功能：部分引脚支持ADC/DAC，用于模拟信号的采集与生成

1.3、工作模式

输入模式
输入模式 特点 补充
上拉输入 引脚内部连接上拉电阻 默认状态稳定(高电平)，外部信号可拉低

上拉电阻越小，驱动能力越强，功耗越高
下拉输入 引脚内部连接下拉电阻 默认低电平，外部信号可将其拉高
浮空输入 引脚内部不连接上拉或下拉电阻 电平完全由外部电路决定

引脚处于高阻抗状态，易受电磁干扰

2. ADC

2.1 概念

ADC（Analog-Digital Converter）模拟-数字转换器

ADC可以将引脚上连续变化的模拟电压转换为内存中存储的数字变量，建立模拟电路到数字电路的桥梁（将模拟信号转换为数字信号）。

STM32的ADC是12位逐次逼近型ADC（工作模式），1us转换时间

2.2 常见 ADC 参数定义

参数 说明
分辨率（Resolution） 指 ADC 能区分的最小模拟信号变化，一般用多少位来表示，每次转换的位数，位数越高，量化结果越精细，对应分辨率越高，分辨率越高，转换后的数字信号越接近原始模拟信号，但电路复杂度和成本也会增加。如 8-bit、12-bit、16-bit 等，决定了电压的最小分辨率（精度）
输入范围（Input Range） 从AD转换开始到产生结果，需要花1us的时间，对应AD转换的频率就是1MHz
转换时间（频率） 支持的模拟电压范围，如 0~3.3V，或 -1V~1V（差分）
精度（Accuracy） 包括 INL（积分非线性）、DNL（微分非线性）、增益误差、偏置误差等
采样率（Sample Rate） 单位时间内的采样次数（如 1Msps 表示百万次/秒）
输入类型 单端输入 vs 差分输入
接口方式 并口、串口（SPI/I2C）、内置 MCU ADC

2.3 ADC架构

ADC有18个输入通道，可测量16个外部信号源和2个内部信号源

16个外部信号：16个GPIO口

2个内部信号：内部温度传感器（CPU温度）、内部参考电压（1.2V的基准电压）

ADC增强单元：规则组（常规事件）和注入组（突发事件）两个转换单元

模拟看门狗自动监测输入电压范围

STM32F103C8T6 ADC资源：ADC1、ADC2，10个外部输入通道。

2.4 规则组的四种转换模式

①单次转换，非扫描模式

每次ADC转换只能选择一个通道并进行一次转换，转换完成后置EOC标志位，如果需要再次转换，就需要重新触发。如果需要换一个通道，则需要将开始的通道换为新通道，然后启动转换。

②连续转换，非扫描模式

每次ADC转换只能选择一个通道，但在一次转换结束后不会停止，而是立刻开启下一轮的转换，也就是说，如果需要再次转换，就不需要重新触发。但是如果需要换一个通道，则需要将开始的通道换为新通道，然后启动转换。

③单次转换，扫描模式

每次ADC转换可以选择多个通道，并依次对多个通道进行转换，转换结果都放在数据寄存器中，为了防止数据覆盖，就需要使用DMA及时将数据挪走，转换完成后置EOC标志位，如果需要再次转换，就需要重新触发。

④连续转换，扫描模式

每次ADC转换可以选择多个通道，并依次对多个通道进行转换，在一次转换结束后不会停止，而是立刻开启下一轮的转换。

2.5 转换时间

AD转换的步骤：采样，保持，量化，编码（ADC逐次比较过程）

STM32 ADC的总转换时间为：

TCONV = 采样时间 + 12.5个ADC周期

例如：当ADCCLK=14MHz，采样时间为1.5个ADC周期

TCONV = 1.5 + 12.5 = 14个ADC周期 = 1μs

2.6 校验

ADC有一个内置自校准模式。校准可大幅减小因内部电容器组的变化而造成的准精度误差。校准期间，在每个电容器上都会计算出一个误差修正码(数字值)，这个码用于消除在随后的转换中每个电容器上产生的误差。

建议在每次上电后执行一次校准。

启动校准前， ADC必须处于关电状态超过至少两个ADC时钟周期。

2.7 电压转换公式

电压（V）= (ADC值 / ADC最大值) * Vref

假设我们的单片机使用的是10位ADC，Vref为5V，读取到的ADC值为512。 那么，电压值计算如下： 电压（V）= (512 / 1023) * 5 ≈ 2.5V

3. DAC

3.1 概念

DAC（数字模拟转换器）是一个关键组件，用于将数字信号转换为模拟信号。

3.2 DAC的引脚配置

  1. VDD/VSS ：这些是电源引脚，用于为DAC提供正电源和地。VDD通常连接到正电源，而VSS连接到地。

  2. Vref ：参考电压引脚。DAC使用这个参考电压来定义其输出范围。例如，如果Vref为3.3V，则DAC的输出范围将是0到3.3V。

  3. Din ：数字输入引脚。这些引脚接收来自微控制器的数字信号，这些信号随后被转换为模拟输出。

  4. Dout ：在某些DAC中，这个引脚可用于级联多个DAC或将转换后的数据输出到其他设备。

  5. Clk ：时钟输入引脚。DAC需要时钟信号来同步数字输入和转换过程。

  6. Sync ：同步引脚，用于同步多个DAC或确保在特定时刻开始转换。

  7. Ldac ：加载DAC寄存器引脚。当此引脚被激活时，数字输入被加载到DAC的内部寄存器中，准备进行转换。

  8. Vout ：模拟输出引脚。这是DAC转换后的模拟信号输出。

3.3 DAC的内部信号

  1. 数字信号 ：这是DAC接收的原始信号，通常是来自微控制器的二进制代码。这些信号在DAC内部被解码和处理。

  2. 寄存器 ：DAC内部有一个或多个寄存器，用于存储待转换的数字值。这些寄存器在时钟信号的控制下更新其内容。

  3. DAC转换器核心 ：这是DAC的实际转换部分，它将存储在寄存器中的数字值转换为相应的模拟电压。这个转换过程是基于一定的算法和内部电路实现的。

  4. 模拟输出缓冲器 ：转换后的模拟信号通常通过一个输出缓冲器来提供驱动能力，以确保稳定的输出和适当的负载驱动。

  5. 控制逻辑 ：DAC内部的控制逻辑负责管理数字输入、时钟信号、同步信号和加载信号等，以确保整个转换过程的正确执行。

3.4 嵌入式开发：DAC的深入细节与实际工作操作

我们概述了DAC（数字模拟转换器）的基本引脚和内部信号。现在，我们将进一步深入探讨DAC的工作原理、配置细节以及与实际工作操作相关的方面。

DAC的工作原理

  1. 数字输入处理 ：

    • 当微控制器或其他数字设备向DAC发送数字信号时，这些信号首先被DAC的数字接口接收。数字接口负责确保信号的稳定性和准确性，以便后续的转换过程。

    • 数字信号可能以并行或串行的方式传输，具体取决于DAC的接口设计。并行传输通常更快，但需要更多的引脚；而串行传输则减少了引脚数量，但可能牺牲了速度。

  2. 寄存器与缓冲 ：

    • 接收到的数字信号被存储在DAC的内部寄存器中。这些寄存器可以是单个或多个，取决于DAC的分辨率和设计。高分辨率DAC通常需要更多的寄存器来存储更多的位数。

    • 为了确保连续且稳定的输出，DAC可能还包含输出缓冲器。输出缓冲器有助于减少输出阻抗，并提供更强的驱动能力，以应对外部负载的变化。

  3. 转换过程 ：

    • 在DAC的核心，数字信号被转换为模拟信号。这通常是通过电阻网络、电流源或电容网络等实现的。不同类型的DAC（如二进制加权电阻DAC、R-2R梯形DAC、电流舵DAC等）采用不同的转换机制。

    • 转换的准确性和线性度取决于DAC的规格和设计。高端DAC通常具有更高的精度和更低的非线性误差。

  4. 模拟输出 ：

    • 转换后的模拟信号从DAC的输出引脚输出。这个模拟信号可以是电压或电流，具体取决于DAC的类型和应用需求。

    • 为了保护DAC免受外部干扰和损坏，输出引脚通常具有保护电路，如过压保护、短路保护等。

DAC的配置细节

  1. 分辨率选择 ：

    • DAC的分辨率是指它能够产生的不同输出级别的数量。例如，一个8位DAC可以提供256个不同的输出级别，而一个12位DAC则可以提供4096个级别。选择合适的分辨率取决于应用对精度和动态范围的需求。

  2. 参考电压设置 ：

    • 参考电压（Vref）对DAC的输出范围和精度至关重要。它必须稳定且准确，以确保DAC输出的准确性。在某些应用中，可能需要使用外部精密参考源来提供Vref。

  3. 时钟与同步 ：

    • 时钟信号对于DAC的操作至关重要，它控制着数字信号的采样率和转换速率。在某些应用中，可能需要与外部设备同步DAC的时钟信号，以确保整个系统的协同工作。

  4. 电源与接地 ：

    • 为DAC提供适当的电源和接地是确保其正常工作的关键。电源噪声和接地回路问题都可能对DAC的性能产生不利影响。因此，在PCB布局和布线时需要特别注意电源和接地的设计。

三、实际工作操作中的考虑

  1. 初始化与校准 ：

    • 在使用DAC之前，通常需要进行初始化和校准步骤。初始化包括设置适当的寄存器值、配置时钟和同步信号等。校准则是为了补偿由于温度、电压变化或制造差异引起的误差。某些DAC提供自动校准功能，可以简化这个过程。

  2. 噪声与失真处理 ：

    • 在实际应用中，DAC的输出可能受到噪声和失真的影响。为了降低这些影响，可以采取措施如使用低噪声电源、优化PCB布局以减少干扰、以及采用适当的滤波技术来平滑输出信号。

  3. 故障检测与处理 ：

    • 在长期运行过程中，DAC可能会出现故障或性能下降的情况。因此，实施故障检测机制是很重要的。这可以通过定期检查DAC的输出、监控其内部状态寄存器或使用外部诊断电路来实现。一旦发现故障，可以采取相应的措施进行修复或重新初始化DAC。

  4. 性能优化 ：

    • 为了获得最佳性能，可能需要根据具体的应用场景对DAC进行优化。例如，调整时钟频率以平衡转换速度和功耗；优化数字输入信号的格式和速率以匹配DAC的输入要求；以及选择合适的外部组件（如滤波器、放大器等）来改善DAC的输出特性。
